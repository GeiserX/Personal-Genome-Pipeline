# CLAUDE.md — Personal Genome Pipeline

## Overview
Whole genome sequencing (WGS) analysis pipeline for consumer hardware. Takes raw FASTQ/BAM/VCF data and runs 34 analysis steps locally in Docker containers: variant calling, pharmacogenomics, structural variants, cancer predisposition, polygenic risk scores, ancestry, telomere length, and more.

## Tech Stack
- Nextflow (pipeline orchestration, `main.nf`)
- Shell/Bash (scripts)
- Docker (all bioinformatics tools containerized)
- nf-core conventions
- Key tools: DeepVariant, minimap2, VEP, PharmCAT, Demucs, WhisperX, CPSR

## Development
```bash
# Run the pipeline
nextflow run main.nf -profile docker --input samples.csv

# Lint
nextflow lint main.nf
```

Requirements: 16+ cores recommended, 500 GB disk per sample, Docker, Nextflow. Runs on Linux, macOS, WSL2.

## Architecture
- `main.nf` — Nextflow entry point
- `conf/` — pipeline configuration profiles
- `modules/` — individual Nextflow process definitions (one per tool)
- `workflows/` — sub-workflow compositions
- `scripts/` — helper scripts
- `assets/` — reference data, gene panels, word lists
- `docs/` — documentation (GitHub Pages)
- `nextflow.config` — base configuration
- `nextflow_schema.json` — parameter schema (nf-core style)
- `versions.env` — pinned tool versions

## Key Rules
- All processing is local; genomic data never leaves the machine
- Pin tool versions in `versions.env`; never use floating tags
- Follow nf-core conventions for module structure
- Reference genome: GRCh38/hg38
